Proxy listeners

Routing required
The explicit proxy listener is the normal forward-proxy endpoint used by browser settings and PAC/WPAD. Optional intercept listeners are for router/firewall redirected HTTP and HTTPS TCP traffic; clients do not configure or receive them in PAC files.
Squid's NAT intercept mode disables proxy authentication on that listener and must be protected from direct external access. Add REDIRECT/DNAT rules on the router or host, exempt the proxy's own source traffic first to avoid loops, redirect TCP/80 to the HTTP intercept port, and redirect TCP/443 to the HTTPS intercept port when enabled.
The HTTPS NAT intercept listener handles redirected TCP/443 TLS only. HTTP/3/QUIC uses UDP/443 and will not traverse Squid's TCP intercept or legacy HTTP proxy path; block or reject UDP/443 at the router/firewall for managed client groups when proxy enforcement matters so clients fall back to HTTP/2 or HTTP/1.1 over TCP.

Web destination ports

Default allow
Non-standard HTTP and HTTPS destination ports are allowed by default. This deployment intentionally does not enforce restrictive Safe_ports or SSL_ports ACLs unless you add them manually.
If you ever need tighter port controls, use the raw Config tab to add explicit ACL restrictions for your environment.
